# Smart Review 0.2.4 多平台内容设计

## 目标

基于 0.1.1 首发文章和 0.2.4 实际功能，制作知乎、公众号、小红书三套独立内容。核心不是重复介绍插件，而是解释它如何从“复习队列”发展为完整的知识复习中心。

## 内容策略

- 知乎：版本演进与产品设计，完整说明新版能力、适用人群和升级价值。
- 公众号：从“知识库只进不出”的问题切入，强调可持续的知识复习闭环。
- 小红书：8–10 页图文卡片，使用短句、强视觉和明确使用步骤。

三篇内容共享事实和真实产品截图，但标题、开头、篇幅、结构和结尾分别适配平台。

## 事实边界

- 当前发布版本为 0.2.4。
- 0.1.1 已有复习队列、四档反馈、状态栏、Markdown 导出、AI Payload 和历史记录。
- 0.2.x 重点新增 Smart Review Center、数据总览、Task Flow、双热力图、分布分析、健康分、中英文和主题适配，以及可配置的动态间隔和实际间隔预览。
- 不把健康分描述为知识质量评分；不宣称插件直接调用 AI 服务。

## 视觉设计

优先采集真实插件界面：中心全景、双热力图、评分间隔预览、设置页。图片统一放入三篇稿件共用的 `assets` 目录。小红书另生成一张竖版封面；功能证明仍以真实截图为主。
